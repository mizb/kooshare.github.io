Zettlr 科研笔记 v3.2.2 中文版

2024-10-07

Zettlr 是款适合写作者和研究人员使用的 Markdown 编辑器，免费开源，功能简洁，具备 Markdown 所有基本功能，内置各种运算符，还可以调用计数器，可以完美替代 Word 和收费的文字处理器。

更新日志

已解决的数据丢失问题 当 Ztelr v3.0.0 发布时，我们开始收到用户的报告 提到一些文件无法正确保存，可能会导致数据 损失。在寻找潜在的根本原因后，我们现在已经将其确定为 文件中不正确的换行符处理。具体来说，我们不小心引入了 这一 bug 将使泽特尔无法正确检测视窗风格 CRLF 换行符。这意味着泽特勒有时只能正确阅读 并修改此类文件。

此更新修复了此 bug。现在，泽特尔能够正确读取和修改任何 文件，无论它是否已在 Windows、macOS、Linux 甚至 一些旧系统。我们想为这个 bug 道歉，并感谢你 尽管如此，还是坚持使用泽特尔。

文件过滤逻辑的更改 文件管理器中的过滤器字段在搜索时始终应用或逻辑 文件和工作区。在这个最新更新中，Ztelr 更改为 AND 文件过滤 逻辑，这意味着只有匹配所有查询的项目才会在 输入以空格分隔的短语。

举个例子：到目前为止，搜索“尼克拉斯·卢曼”会出现文件 包含“尼克拉斯”或“卢曼”，或两者兼而有之。从现在开始，搜索 “尼克拉斯·卢曼”只会显示包含“尼克拉斯”和“尼克拉斯”的文件 “Luhmann”并排除错过这些短语之一的文件。

GUI 和功能 特点：附件/资产/其他文件侧边栏选项卡现在也显示文件 在适用的默认图像文件夹中找到 特点：现在右键单击外部降价链接的上下文菜单 包含删除链接的选项。删除样式链接时， link 文本保持为纯文本。删除title样式链接时， 该 title 文本保持为纯文本。 更改：在过滤器字段中搜索文件时，只有文件和 将显示与输入的所有查询匹配的工作区 修复了未保存更改对话框操作的法语翻译。（#5177） 修复了在 Windows 上正确保存文件（和保留换行符）的错误 系统；现在 Ztelr 应该能够处理任何类型的行（#5109） 修复了各种列表控件中的复选框不正确的问题 更新以反映实际的基础价值 修复侧边栏中的资产文件图标 侧边栏中的设计修复 修复：文件预览工具提示现在尊重文件名显示设置 修复：在文件夹中搜索（全局搜索）时聚焦输入字段 触发 兜帽之下 升级 Electron 到 v30.1.0（参见问题#5135 和 Electron 问题#41839） 降级 Linux 构建以使用 Ubuntu 20.04 而不是 22.04（#5137） 从内部逻辑中完全抽象出换行符处理。现在，换行符 始终\n 在整个应用程序中。文件中的实际换行符将 存储在各自的文件描述符中，并将专门用于 文件读取（将它们替换为\n）和文件写入（将\n 替换为）
