WizTree 4.20 (大文件查找) 单文件

2024-08-15

查找大文件WizTree，用来寻找占用大量分区的无用文件和文件夹，找出占用空间的大文件的小工具，现在也可以扫描非NTFS硬盘驱动器，网络驱动器，USB驱动器和单个目录，和everything一样的工作原理。

软件特色

• 界面中会使用目录树方式把文件夹从大到小排列，并且还会显示占用百分比、占用空间、文件数量、子文件夹数量等信息 • “top 1000 largest files”标签页中还会显示体积最大的1000个文件　　 • 你可以在界面中进行删除、复制等操作，和资源管理器无异 • 完全免费。查找大文件使用方法运行后，选择某个分区，点击“scan”按钮，等待几秒就可以看到结果了，扫描使用的时间在扫描结束后也可以在界面上看到。而对分区格式的话，WizTree有一些要求，那就是只能扫描NTFS格式的分区

v4.20 版本更新：2024年7月29日

– 添加了新的调整选项（选项->调整）： “在选择用于分析的文件夹时扫描 MFT（如果可能）” – 默认值为关闭 通常，当选择一个文件夹进行分析时，WizTree 将使用 Windows API 进行扫描（不使用 MFT 扫描）。在分析中小型文件夹时，此方法通常更快，但对于非常大的文件夹，扫描整个 MFT 文件可能会更快。 如果启用此选项，则在选择文件夹时，WizTree 将始终扫描 MFT 文件（可能）。您还可以通过在开始新扫描时按住 CTRL 键来强制 WizTree 在选择文件夹时扫描 MFT。扫描过程开始时，必须按住 CTRL 键才能正常工作。 – 修复了扫描多个驱动器/文件夹时文件视图中可能出现的顺序倒序排序问题。 – 在 Windows XP/2000/2003 上再次将文件剪切/复制到剪贴板（由于与新代码的兼容性问题，在 4.18 中停止工作）。 – 处理包含数百万个具有唯一文件扩展名文件的文件夹大大加快了速度。以前，当将此类文件夹加载到树视图中或生成树状图时，WizTree 可能会停止响应。 – 树状图导出现在始终按大小顺序处理文件和文件夹，因为这样生成的输出看起来要好得多。 – WizTree 的 Windows 资源管理器上下文菜单快捷键从 W 更改为 Z（从 &WizTree 更改为Wi&zTree），以防止与“新建”选项冲突。 – 修复了在某些情况下，硬链接文件的文件大小/分配大小可能错误地设置为 0 的问题。 – 在某些情况下，MFT 扫描速度比以前的版本快 15%。
